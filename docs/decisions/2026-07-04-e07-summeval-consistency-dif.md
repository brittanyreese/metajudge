# E07 real-data demo: SummEval consistency/fluency/relevance DIF

Date: 2026-07-04 Status: accepted (worked example; not a claim that SummEval's expert panel is factually miscalibrated in general) Script: `examples/audit_summeval_consistency.py` Data: SummEval (Fabbri et al., 2020), same expert panel already used by `examples/audit_summeval.py` (MIT, corpus lock: `docs/decisions/2026-06-22-e07-corpus-lock-summeval.md`)

## Why this run

`examples/audit_summeval.py` audits SummEval's coherence dimension and lands a clean class A null (Nagelkerke R2 delta 0.002, p_uniform = 0.0005 but the effect size is negligible). That is a fine first demo but a soft one: the DIF pillar never gets to show a decisive statistical hit on this corpus. Coherence also is not the dimension with the strongest a-priori reason to differ by system family. This run asks the same extractive-vs-abstractive question of the other three annotation dimensions, chosen and ranked before any of them were scored against the panel.

## Design, pre-registered before this script was ever run

Decided and written down before `logistic_dif` was ever called on these three dimensions:

- **Dimensions and rank**: consistency (PRIMARY), fluency (SECONDARY), relevance (EXPLORATORY). Consistency is primary because extractive summaries lift sentences verbatim from the source, so factual consistency is close to guaranteed by construction, while abstractive generation can hallucinate content the source does not support. That asymmetry gives consistency the strongest a-priori reason, of the three, to differ between the two system families. Fluency is secondary (abstractive models can produce less fluent text when generation goes wrong, but the SummEval systems are all pretrained language models, so the gap is a weaker prior than consistency's). Relevance is exploratory: no strong direction was predicted before running it. All three were named and ranked together, so this is one family of three pre-registered tests, not a post-hoc sweep across dimensions that looked promising.
- **Item**: summary (`{article_id}_{model_id}`), same convention as `audit_summeval.py`.
- **Rater**: `expert_0`/`expert_1`/`expert_2`, the 3-expert panel's slot labels (`scripts/prep_demo.py`).
- **Stratum**: system family. focal = abstractive (n=900 summaries), reference = extractive (n=700 summaries), the same pair as `audit_summeval.py`.
- **Conditioner**: external, leave-one-dimension-out (LODO). For the dimension under audit, the per-item mean over the 3 experts of the OTHER three annotation dimensions (SummEval scores 4: coherence, consistency, fluency, relevance). This is independent of the studied dimension's own score by construction, which is what an external DIF conditioner needs (`docs/decisions/2026-06-22-e07-dif-ordinal-logistic-regression.md`). It is a stronger match than `audit_summeval.py`'s default rest-score conditioner, which cannot see a bias the whole 3-expert panel shares.
- **Multiplicity**: Holm-Bonferroni correction (`holm_adjust`) across the 3-dimension family's `p_total` values.

## Data

`examples/data/summeval_dims.csv`, generated by `scripts/prep_demo.py --jsonl <path> --multi-dim` from the same SummEval `model_annotations.aligned.jsonl` `audit_summeval.py` reads (MIT license, not vendored; see `src/metajudge/data/SOURCE.md`). 4,800 rows: 1,600 summaries (100 articles x 16 systems) x 3 expert raters, with all four annotation dimensions as columns (`item, rater, stratum, coherence, consistency, fluency, relevance`) so the LODO conditioner above is reproducible from the committed CSV alone, without re-parsing the raw JSONL.

Per-stratum item N: abstractive 900, extractive 700, both comfortably clearing the >=500 Jodoin-Gierl calibration floor (`docs/decisions/2026-06-22-e07-dif-ordinal-logistic-regression.md`).

## Results

All three run with n_obs = 4,800 (3 raters x 1,600 items), external LODO conditioner, `abstractive` vs `extractive`:

| Dimension | Role | chi2 uniform | p uniform | chi2 nonuniform | p nonuniform | chi2 total | p total (Holm) | R2 delta | Class | Cond. corr | Overlap weak |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| **Consistency** | Primary | 1.72 | 0.190 | 56.23 | 6.5e-14 | 57.95 | ~0 | 0.0146 | **A** | 0.153 | False |
| Fluency | Secondary | 2.54 | 0.111 | 9.10 | 0.0026 | 11.64 | 0.0030 | 0.0028 | A | 0.158 | False |
| Relevance | Exploratory | 12.28 | 0.0005 | 2.51 | 0.114 | 14.79 | 0.0012 | 0.0025 | A | 0.135 | False |

Cluster-robust check on the primary dimension (item-cluster bootstrap, n_boot=400, seed=7): consistency's Nagelkerke R2-delta 95% CI is [0.0063, 0.0295], percentile method, all 400 resamples effective. The CI sits entirely inside the Jodoin-Gierl negligible band (below 0.035): the cluster-robust check does not overturn the class A read, it confirms it.

**All three results converged and all classify as Jodoin-Gierl class A (negligible).** The headline is the primary result's split between significance and magnitude: consistency's nonuniform-DIF likelihood-ratio test is decisive (chi2=56.23, p=6.5e-14, Holm-adjusted p is still effectively 0 after correcting for the 3-test family), two orders of magnitude past the shipped coherence demo's p=0.0005. But the Nagelkerke R2-delta effect size is 0.0146, under half the 0.035 negligible threshold and the cluster-robust CI does not reach it either. At n_obs=4,800 the analytic test has power to detect an effect size with no practical consequence: exactly the significance-is-not-magnitude framing the report card already leads with (`docs/decisions/2026-07-02-e07-overlap-threshold-calibration.md`), now demonstrated on a p-value that clears significance by 11 orders of magnitude rather than by 2.

The nonuniform (interaction) term drives consistency's result, not the uniform (main-effect) term: p_uniform = 0.190, not significant on its own. Interpreted substantively, this says the abstractive-vs-extractive consistency gap is not a flat offset across the quality range; it is concentrated at particular points on the quality axis (the interaction between group and the LODO conditioner). A flat main-effect shift would have shown up as uniform DIF instead.

## Whether the audit caught consequential DIF

Not this time, and that is itself informative. The pre-registered primary analysis (consistency) is exactly the dimension SummEval's own construction makes it a-priori plausible for: extractive systems are almost immune to inconsistency by definition, abstractive systems are not. The audit found a statistically decisive difference in how the two families' consistency scores relate to the rest of the rubric (the nonuniform term), but the size of that difference is small on the Jodoin-Gierl scale, and a 400-resample cluster-robust bootstrap agrees. Fluency and relevance, run in the same pre-registered family, land in the same place: significant somewhere in the total chi-square, negligible in R2-delta.

Read together with `audit_summeval.py`'s coherence null, four of SummEval's four annotation dimensions now have a DIF read: three show the same "significant, negligible" pattern (coherence, fluency, relevance) and one (consistency) shows a sharper version of it (a decisive nonuniform term). None crosses into Jodoin-Gierl class B or C. That is evidence the 3-expert panel's family-conditional scoring, on this corpus, does not carry a substantively large DIF signal, at least not one visible to a per-dimension LODO conditioner. It is not evidence that no bias exists: see caveats below, particularly the conditioner-independence limit.

## Caveats

1. **The LODO conditioner is not fully independent.** All four SummEval dimensions are scored by the SAME 3 experts. A leave-one-dimension-out conditioner is independent of the STUDIED SCORE (by construction, since it excludes that dimension), but it is not independent of the RATER PANEL: if all 3 experts share a correlated blind spot across every dimension (a shared halo effect, or a shared tendency to rate abstractive summaries generously across the board), the LODO conditioner would be biased in the same direction as the studied dimension and could under- or over-state the true DIF either way. This is the same instrument-level caveat `audit_ellipse.py`'s findings doc raises for its own leave-one-trait-out conditioner (`docs/decisions/2026-07-04-e07-ellipse-human-rater-dif.md`), and the same nested-strata confound the report card always carries (`docs/decisions/2026-07-01-e07-dif-nested-strata-confound.md`): read every number above as screening evidence for the pre-registered family, not a confound-free instrument-level fairness clearance.
2. **Small item-level N per stratum for a bootstrap.** 900 focal / 700 reference items clear the >=500 Jodoin-Gierl floor comfortably, but the cluster CI in the results table was computed for consistency only; fluency and relevance were not given a bootstrap pass in the committed script (running all three at n_boot=400 takes several minutes; see the performance note below). The point estimates for fluency and relevance should be read as analytic-only.
3. **Statistical significance is doing a lot of the narrative work here.** The whole point of this demo is that "significant" and "large" are different questions. A reader skimming only the p-value column would misread this run as three fairness problems; the R2-delta column is what actually answers whether any of them warrants action, and none does.
4. **conditioner_group_corr is inside the safe band for all three (0.135-0.158, well under the 0.2 overlap-weak threshold, `docs/decisions/2026-07-02-e07-overlap-threshold-calibration.md`)**, so none of these results is a confound artifact of the conditioner correlating with group membership; the negligible-magnitude read is not an overlap problem in disguise.
5. **Performance.** `audit(robust=True, n_boot=400)` on this corpus's 4,800-observation pooled fit takes roughly two minutes per dimension on a laptop; the committed example script runs all three dimensions with the bootstrap enabled, so a full run takes several minutes. The committed test suite pins the fast analytic-only path (`logistic_dif`, no bootstrap) by default; the cluster-bootstrap-CI pin and the full-script output-drift guard are both marked `@pytest.mark.slow` and run only under `--run-slow` (the existing Rigor-workflow convention, `docs/decisions/2026-07-03-ci-slow-suite-performance.md`), so `uv run pytest` stays fast.
