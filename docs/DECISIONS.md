# Decisions

Every choice that changes the build is recorded as a dated, cited ADR in [decisions/](decisions/). This is the curated index: what was decided and why, newest first. The why-this-build context is in [PROVENANCE.md](PROVENANCE.md); the requirements those decisions feed are in [SPEC.md](../SPEC.md).

## E07 report card

CI: parallelize the Rigor slow suite, accept the two-core ceiling (2026-07-03). The slow leg (operating-characteristics cells plus the live R oracle refit) ran 77 minutes serially and gates every release twice. The replications are independently seeded, so `pytest-xdist -n auto` distributes them bit-identically, but on the two-core runner numpy and scipy already use both cores through OpenBLAS, so the workers oversubscribed and gained almost nothing until the BLAS thread count was pinned to 1. That brings the gate to 58 minutes. A larger runner would go faster but bills per minute and breaks CI for anyone who forks the repository, so the build stays on the standard runner and the speed left on the table is recorded rather than taken. [ADR](decisions/2026-07-03-ci-slow-suite-performance.md)

Conditioner-overlap flag recalibrated from convention to measurement (2026-07-02). The operating-characteristics study ([sim-operating-characteristics.md](sim-operating-characteristics.md)) measured the false Jodoin-Gierl B/C rate under a true null against the correlation each run observes: 0.5% at |corr| <= 0.2, 17% in (0.2, 0.4], above 50% past 0.4, while the original 0.7 convention almost never fired even in cells with a 100% false-DIF rate. `_OVERLAP_WEAK_CORR` moved from 0.7 to 0.2 and the card's warning now cites the measured band. The same study reports baseline Type-I/power, the absence of analytic inflation in this crossed design, PO-violation robustness, and the power curve. [ADR](decisions/2026-07-02-e07-overlap-threshold-calibration.md)

Nested-strata conditioner/group confound is a stated scope limit, not a bug (2026-07-01). "DIF across output strata" inverts classical DIF: group is a property of the item, not of an examinee answering a shared item, so the conditioner is matched between two disjoint item sets rather than within shared items. When the strata genuinely differ in quality, the conditioner correlates with the group and the single linear match leaves residual confounding. `DifResult` carries `conditioner_group_corr`, `conditioner_common_support`, and `conditioner_overlap_weak` so the report card can name the per-run regime instead of a blanket caveat. [ADR](decisions/2026-07-01-e07-dif-nested-strata-confound.md)

Report card: clustering-robust DIF flag, not raw analytic p-values (2026-07-01). The analytic uniform/nonuniform p-values are anti-conservative under the crossed rater x item design and fed no decision the card rendered, so the card leads with the effect size and A/B/C class and replaces the significance headline with a clustering-robust flag derived from the item-cluster bootstrap; the analytic lines stay but are tagged `[analytic, unclustered]`. [ADR](decisions/2026-07-01-e07-report-card-inference-caveat.md)

DIF: cluster bootstrap for clustered-error inference (2026-06-23). `logistic_dif` pools each (item, rater) cell as independent, which is anti-conservative under the crossed rater-by-item design. `cluster_bootstrap_dif` adds a non-parametric item-cluster bootstrap (items resampled with replacement, stratified within focal/reference, full rater blocks kept intact) as a robustness layer on top of the unchanged analytic engine. [ADR](decisions/2026-06-23-e07-dif-cluster-bootstrap.md)

Eval-instruments interop: map judge output to Ratings with rater = judge (2026-06-23). `Ratings.from_eval_instruments` ingests one `frame_from_evals` output per judge (Epic's evaluation-instruments) under the frame rater = judge, item = sample, score = one rubric criterion. Criteria are a separate facet, audited one at a time, never raters; a criteria-as-raters frame would measure internal consistency (Cronbach), not inter-rater reliability. A method-currency pass and adversarial review validated the frame against the rater-mediated-measurement literature. No dependency is added. [ADR](decisions/2026-06-23-e07-eval-instruments-interop.md)

ICC on incomplete data: refuse and name the right estimator (2026-06-23). Shrout-Fleiss ICC(2,1)/(2,k) is defined on a complete crossed design. For a matrix with missing cells, the correct estimand is a variance-components estimator (ten Hove et al. 2024), which is outside this two-pillar package, so `icc` refuses rather than returns a biased listwise number, and the error names the correct method. A method-currency pass and adversarial review rejected building a lightweight irrNA/Ebel estimator now: its only oracle is too thin for a non-compressible reliability number, and the machinery is full-instrument scope. [ADR](decisions/2026-06-23-e07-icc-incomplete-data-refuse.md)

DIF engine: ordinal logistic regression, not Mantel-Haenszel (2026-06-22). The dichotomized Mantel-Haenszel pillar matched each unit on its own per-rater mean, so the matching variable contained the studied response and the analysis was circular (the verdict swung with the bin count). It is replaced by proportional-odds logistic-regression DIF (the Zumbo and lordif framework) conditioned on an independent leave-one-rater-out rest score, with the Nagelkerke R-squared change classified by the Jodoin-Gierl thresholds. [ADR](decisions/2026-06-22-e07-dif-ordinal-logistic-regression.md)

Demo corpus lock: SummEval expert coherence (2026-06-22). The worked demo uses the SummEval expert-coherence subset, a real crossed subject-by-rater matrix that computes alpha, ICC, and DIF without generating any data. It was chosen over HealthBench because it ships a crossed structure with per-rater scores, so data sourcing never blocks the correctness work. [ADR](decisions/2026-06-22-e07-corpus-lock-summeval.md)

The two-pillar method choices rest on a literature review of reliability, DIF, and rater-mediated-measurement methods; the sources are consolidated in [REFERENCES.md](REFERENCES.md). A research finding graduates into an ADR here only when it changes the build.
